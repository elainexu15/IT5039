GitHub Instructions

why do we use git?

git allows us to track the changes that we make to our code in order for us to be able to revert to older versions if there's a problem. In addition, when we work with teams, many people collaborate on one large code base, git helps us manage the changes that are made.

Are git and GitHub the same thing?

It is important to know that git and GitHub are not the same thing. git is a version control system that allows us to track changes that we make to code. Github is a hub for git repositories, in other words, it is a space where we store our git repositories.

How to check if you have git installed properly?

open vscode, click view -> terminal to open up the terminal. in the terminal, make sure you have powershell open. type command git --version and hit enter. if you have git installed, it will give you the version number of git that is currently installed.
If you want to know where git is installed, you can type command where.exe git and hit enter. it will give you the path to the executable file of git.
If git --version command doesn't return a version number, then you need to download and install git first.

how to solve error: git remote upstream already exists.

If you have two github account, you may encounter this error message: git remote upstream already exists.
If you're on Windows, you can remove the stored Git/GitHub credentials from Control Panel > User Accounts > Credential Manager.
